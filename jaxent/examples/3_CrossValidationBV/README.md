# Example 3 — Cross-Validation with BV Regularisation

## Overview

This example extends [Example 2 (Cross-Validation)](../2_CrossValidation/README.md) by introducing **physical regularisation on the Best–Vendruscolo (BV) model parameters** alongside Maximum-Entropy (MaxEnt) ensemble reweighting. The result is a **2D hyperparameter sweep** over:

| Axis | Parameter | Purpose |
|------|-----------|---------|
| **X** | MaxEnt scaling (`λ_MaxEnt`) | Controls the strength of the KL-divergence penalty that keeps frame weights close to a uniform prior |
| **Y** | BV regularisation scaling (`λ_BV`) | Penalises the BV contact parameters (`β_c`, `β_h`) via an L1 or L2 norm to prevent over-fitting to the structural data |

By sweeping both axes simultaneously the pipeline identifies the optimal trade-off between data fidelity, ensemble diversity (MaxEnt), and physical model robustness (BV regularisation).

### Scientific Motivation

Standard ensemble reweighting adjusts frame weights to improve agreement with experimental HDX-MS data. However, simultaneously optimising the BV model parameters (`β_c` for heavy-atom contacts, `β_h` for hydrogen-bond acceptor contacts) can lead to over-fitting — the parameters absorb noise rather than capturing genuine physical effects. Adding a regularisation penalty on these parameters constrains the optimisation landscape and improves generalisation, as assessed through cross-validation.

### Biological System

The system under study is **Mouse Prion Protein (MoPrP)**, a partially disordered protein that populates multiple conformational states:

| State | Description |
|-------|-------------|
| **Folded** | Native globular domain |
| **PUF1** | Partially Unfolded Form 1 |
| **PUF2** | Partially Unfolded Form 2 |
| **Unfolded** | Fully disordered |

Target state populations are defined in `state_ratios.json` and serve as the ground truth for recovery analysis.

### Ensembles

| Ensemble | Description |
|----------|-------------|
| `AF2_MSAss` | AlphaFold 2 MSA-subsampled ensemble (GT+alt — contains both ground-truth and alternative conformations) |
| `AF2_filtered` | AlphaFold 2 filtered ensemble (GT-only — enriched for ground-truth conformations) |

## Workflow

The workflow has three phases: **Setup**, **Fitting**, and **Analysis**.

### Phase 1 — Data Setup

Copy the required data from Example 2:

```bash
# From the 3_CrossValidationBV directory:
cp -r ../2_CrossValidation/fitting/jaxENT/_datasplits     fitting/jaxENT/
cp -r ../2_CrossValidation/fitting/jaxENT/_featurise       fitting/jaxENT/
cp -r ../2_CrossValidation/fitting/jaxENT/_MoPrP_covariance_matrices fitting/jaxENT/
cp    ../2_CrossValidation/analysis/state_ratios.json      analysis/
```

### Phase 2 — Model Fitting (2D Hyperparameter Sweep)

#### Quick Test

```bash
cd fitting/jaxENT
bash run_maxent_parallel_BV_objective_test.sh
```

This launches a reduced sweep for rapid validation. The test script uses fewer MaxEnt/BV-reg values and fewer optimisation steps.

#### Full Sweep

```bash
bash run_maxent_parallel_BV_objective.sh \
  --ensembles "AF2_MSAss,AF2_filtered" \
  --losses "mcMSE,MSE,Sigma_MSE" \
  --split-types "random,sequence_cluster,spatial" \
  --maxent-values "0.0,0.1,1.0,10.0,100.0" \
  --bvreg-values "0.0,0.01,0.1,1.0,10.0" \
  --bvreg-loss "L1,L2" \
  --n-steps 10000 \
  --num-splits 3
```

#### What the Sweep Does

For each combination of `(ensemble, loss_fn, split_type, maxent_value, bvreg_value, bvreg_loss)`:

1. **Loads** pre-computed BV features and the corresponding train/val data split
2. **Constructs** a `Simulation` object with the BV forward model
3. **Optimises** frame weights and BV parameters (`β_c`, `β_h`) using the configured loss function, MaxEnt penalty, and BV regularisation penalty
4. **Saves** the optimisation history (parameters, losses per epoch) to an HDF5 file

#### Loss Functions

| Name | Description |
|------|-------------|
| `mcMSE` | Mean-centred MSE — subtracts per-peptide mean before squaring |
| `MSE` | Standard Mean Squared Error on deuterium fractions |
| `Sigma_MSE` | Covariance-weighted MSE using experimental uncertainty matrices |

#### BV Regularisation Types

| Name | Penalty |
|------|---------|
| `L1` | `λ_BV × (|β_c| + |β_h|)` — promotes sparsity |
| `L2` | `λ_BV × (β_c² + β_h²)` — promotes small parameters |

#### Data Split Strategies

| Strategy | Description |
|----------|-------------|
| `random` | Random peptide assignment |
| `sequence_cluster` | Non-redundant: clusters peptides by sequence overlap |
| `spatial` | Clusters peptides by 3D spatial proximity |

### Phase 3 — Analysis Pipeline

The analysis pipeline consists of up to eight sequential steps. Each step builds on the outputs of the previous one.

#### Step 1: Preliminary Recovery Analysis

```bash
python ../analysis/recovery_analysis_ISO_TRI_BI_2D_BV.py \
  --results-dir <results_dir> --clustering-dir <clustering_dir>
```

Generates **2D heatmaps** of conformational state recovery (%) over the (MaxEnt, BV-reg) grid. Recovery is computed using `1 − √JSD`, where JSD is the Jensen–Shannon Divergence between the reweighted state populations and the target populations from `state_ratios.json`.

**Outputs:** Recovery heatmaps (per ensemble × loss × split type), 1D slice plots.

#### Step 2: Weights Validation

```bash
python ../analysis/weights_validation_ISO_TRI_BI_2D_BV.py \
  --results-dir <results_dir>
```

Evaluates the quality and consistency of the optimised frame weights:

| Metric | Description |
|--------|-------------|
| **KL Divergence** | Distance of frame weights from the uniform prior |
| **ESS** | Effective Sample Size — measures how many frames contribute meaningfully |
| **Pairwise KLD** | Agreement between weights from different cross-validation splits |

**Outputs:** 2D heatmaps of KLD, ESS, and pairwise KLD across the hyperparameter grid.

#### Step 3: Loss Analysis & Model Selection

```bash
python ../analysis/analyse_loss_ISO_TRI_BI_2D_BV.py \
  --results-dir <results_dir>
```

Analyses the optimisation trajectories and performs automated model selection:
- Convergence heatmaps across the 2D grid
- Model score (validation loss) heatmaps
- Best-model identification per split type

**Outputs:** Convergence heatmaps, model score heatmaps, best-model summary tables.

#### Step 4: Process Optimisation Results

```bash
python process_optimisation_results.py \
  --results-dir <results_dir> \
  --clustering-dir <clustering_dir> \
  --features-dir <features_dir> \
  --datasplit-dir <datasplit_dir>
```

Extracts detailed predictions from all HDF5 optimisation histories:
- Per-residue log protection factors (`lnPF`)
- Peptide-level uptake predictions
- Frame weights and KL divergence from uniform prior
- Cluster population ratios
- Optimised BV parameters (`β_c`, `β_h`)

**Outputs:** `.npy` files (predictions, weights, parameters) and `.csv` files (cluster ratios) per run, organised in a `_processed_*` directory.

#### Step 5: Score Models

```bash
python score_models_ISO_TRI_BI.py \
  --processed-data-dir <processed_dir> \
  --datasplit-dir <datasplit_dir> \
  --features-dir <features_dir> \
  --clustering-dir <clustering_dir> \
  --state-ratios-json state_ratios.json
```

Computes comprehensive quality metrics for every optimisation run:

| Metric Category | Metrics |
|-----------------|---------|
| **Error** | Train/Val/Test MSE, dMSE (improvement over prior) |
| **Thermodynamic Work** | Work Scale, Shape, Density, Fitting, Magnitude |
| **Reweighting** | KL divergence from uniform prior |
| **Recovery** | Ground-truth state recovery percentage (1 − √JSD) |

**Outputs:** `model_scores.csv` containing all metrics, plus violin plots.

#### Step 6: Statistical Modelling

```bash
python analyse_scores_mixed_linear_model.py \
  --scores-csv-path <scores_csv> \
  --target-metric recovery_percent
```

Performs rigorous statistical analysis of what drives ground-truth recovery:
- **Multiple regression**: `recovery = f(MSE, Work metrics, KL, ...)`
- **Stability analysis**: consistency of each metric across data splits (ANOVA, eta-squared)
- **Mixed-effects models**: quantifies variance attributable to split type vs. random effects
- **ICC (Intra-Class Correlation)** summaries

**Outputs:** Regression coefficient tables, stability summaries, partial R² comparisons, scatter plots.

#### Step 7: Visualise Model Selection Performance

```bash
python plot_selected_models_ISO_TRI_BI.py \
  --before-path <unfiltered_summary.csv> \
  --after-path <filtered_summary.csv>
```

Creates publication-ready visualisations:
- Recovery score bar plots (faceted by split type, with ensemble comparison)
- Statistical significance (Welch's t-test) annotations between ensembles
- Before/after convergence filtering comparisons
- Performance vs. Inconsistency scatter and bar charts
- Minimax recovery panels (minimum of mean recovery across losses)
- Kendall's W concordance across ground-truth scores

**Outputs:** PNG plots for each visualisation type.

#### Step 8: Cross-Experiment Analysis (Optional)

```bash
python cross_experiment_mixed_effects_model.py
```

Integrates results across multiple JAX-ENT experiments (Example 1 IsoValidation, Example 2 Reweighting, Example 3 RW+BV) to:
- Assess metric robustness across diverse systems using linear mixed-effects models
- Quantify variance components (fixed: metric, class, experiment, loss, split; random: ensemble)
- Generate performance vs. robustness scatter plots and 3D summaries
- Identify the most stable and predictive indicators

**Outputs:** Variance decomposition plots, forest plots, interaction heatmaps, robustness summaries.

#### Comparison with HDXer (Optional)

```bash
python plot_compare_jaxENT_HDXer.py
```

Creates harmonised comparison bar plots between HDXer and JAX-ENT results, panelled by ensemble and split type, showing ground-truth recovery and KL divergence.

## Key Metrics

### Recovery Percentage

$$\text{Recovery}(\%) = 100 \times \left(1 - \sqrt{\text{JSD}(P \| Q)}\right)$$

where $P$ is the reweighted state distribution and $Q$ is the target distribution from `state_ratios.json`.

### dMSE (Delta MSE)

$$\text{dMSE} = \text{MSE}_{\text{posterior}} - \text{MSE}_{\text{prior}}$$

Negative values indicate the optimised model improves over the uniform-weight prior.

### Thermodynamic Work Metrics

Derived from the change in log protection factors between prior and posterior:

| Metric | Interpretation |
|--------|---------------|
| **Work Scale** | Global shift in protection factor magnitude |
| **Work Shape** | Energetic cost of changing the relative PF profile |
| **Work Density** | Entropic cost of redistributing protection factors |
| **Work Fitting** | Total thermodynamic work done during optimisation |

### Effective Sample Size (ESS)

$$\text{ESS} = \frac{1}{\sum_i w_i^2}$$

where $w_i$ are the normalised frame weights. Higher ESS indicates a more diverse ensemble.
